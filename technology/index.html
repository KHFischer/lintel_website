TECHNOLOGY

The verification stack

Lintel's verification stack is built from four components: a specification language, sealed execution hardware, a cryptographic output format, and a chaining mechanism that binds outputs over time. Each component is standard; the composition is new.

SPECIFICATION

The Open Veracity Language

Strategies are specified in the Open Veracity Language — a small, declarative language of roughly 25 mathematical primitives. OVL is deliberately restricted: it expresses what a strategy should compute, not arbitrary code that executes it. The restriction is what makes verification tractable. The language is published and implementable by any party; only Lintel's implementation is proprietary.

EXECUTION

Sealed execution

The OVL specification is loaded into a Trusted Execution Environment — commodity confidential-computing hardware (AMD SEV-SNP, Intel SGX, Intel TDX) that cryptographically isolates the computation. The strategy and market data enter the sealed environment, computation completes, and the environment is wiped. Nobody outside the hardware — including Lintel — can observe or extract the logic. Confidentiality is enforced by the silicon, not by policy.

OUTPUT

The Strategy Verification Package

Execution produces a Strategy Verification Package — a tamper-evident record that binds the claimed results to the specific computation that produced them. The package carries a hardware attestation proving which code ran on which data inside which environment. Any modification after the fact breaks the cryptographic seal and is detected on verification. The SVP is the artifact allocators actually consume; everything else is infrastructure.

LINEAGE

Chained verification over time

SVPs chain over time. Each verified run references the previous one, so a manager's record becomes a continuous, cryptographically-linked lineage rather than a series of disconnected snapshots. An allocator can verify not just that one result is genuine, but that the record is unbroken. Any gap, substitution, or restart is detectable by construction. Point-in-time audits cannot provide this property.
